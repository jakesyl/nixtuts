NixTuts stuff

Q: What does "NixTuts" mean?

A: *nix Tutorias in full form :D We refer to tutorials as "tuts" too. *nix is commonly used in IRC channels as a reference to Unix-like systems.

Q: Why do you have tutorials (CSS, JS etc.) that aren't related to *nix?

A: Many *nix users want to learn those too so we included them. :)

Q: What's your ultimate goal?

A: To help out the *nix community through providing knowledge to those who are eager for it, and help out poor develoeprs working tirelessly if possible.

IRC

Q: Sooooo many channels? Which one?

A: Choose the channel that is in a server that you use. If no such server exists, join the main channel.

Q: I've been banned from the channels! What should I do?

A: We usually ban people with a reason and thus if you are banned you should see a reason. Any ban without a reason is considered invalid and will be removed. If you still have a reason... try entering other channels and explaining your reason. But keep it civil, else we'll ban you. If we really don't want you to enter, look for 1) an abnormal ban that causes auto kick 2) widespread ban. In most cases this will not happen.

Contributing to NixTuts

Q: How could I contribute to NixTuts?

A: We're currently working on a new tutorial for NixTuts contribution.

Q: I've edited the site with the link specified at the bottom of the page. When will they be added?

A: A collaborator already part of the team will review your pull request and decide accordingly. We might also add you into the team (and humans.txt file).

Q: How does the link below work?

A: The link below is a link to prose.io, an online GitHub site editor. Depending on your GitHub account, if you're not part of the team it'll send a pull request when you update the page. How pull requests work is sort of creating your own repository, editing it, then requesting the official team to accept your change. If we do accept your change, you might be added to the team :)

If you're already part of the team, the commit will be pushed straight away and changes should show within 5 minutes.

Misc.

Q: Why not use a CMS?

A: We *used* to use a CMS, but we've changed to a static convertor, or BinPress (our version of Jekyll that has slight changes). Static convertors require no PHP, thus improving speed of website, being less buggy, less exploitable, and more maintanable :) We also have much more control over how our site looks like etc.

Q: What host are you using?

A: We're using Subsentient's server, universe2.us, and use CloudFlare too to speed up our site and provide security, as well as reduce load.

Q: How did you make the site auto-updating?

A: We use bash scripts to make our life easy. First, we have two sections: one is the Git repository, hosting text files; the other one is other files to be included in the site. The script first checks if the Git repository is up-to-date and if the binary files have changed with a checksum. Then, if something has changed, the script updates the site. It builds the Git repository with Jekyll, and copies the built site + binary files over. The script runs periodically every minute, ensuring the site is always up to date.
